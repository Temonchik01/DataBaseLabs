# Лабораторна робота №1 Виконали: Стіренко Тимофій та Клименко Андрій
В цій лабораторній роботі ми продемонструємо реалізацію системи створення персонажів для Turn-Based Videogame через ER diagram 
## Вимоги
**Мета:** Створити гнучку реляційну структуру для керування персонажами, їхніми станами під час бою, а також уніфіковану систему атак та предметів.
-   **Потреби зацікавлених сторін:**
    -   **Геймдизайнери:** Можливість легко створювати нові типи атак та предметів, комбінуючи їх з різними "готовими" ефектами.
    -   **Розробники:** Чітке розмежування між "шаблоном" персонажа (статичні дані) та його конкретним екземпляром у бою (динамічні дані).
-   **Дані для зберігання:** Характеристики персонажів (ОЗ, атака, захист), інвентар (предмети), список доступних атак та механіка накладання ефектів.
-   **Бізнес-правила:**
    -   Персонаж у бою базується на існуючому шаблоні (BaseCharacter).
    -   Атаки та предмети можуть мати один або декілька ефектів.
    -   Один боєць може володіти кількома атаками одночасно.
## Список сутностей та опис зв'язків
### Сутності та атрибути:
-   **BaseCharacter (Шаблон):** Містить базові ліміти персонажа (`max_hp`, `max_attack`, `max_speed`). Це дані класу героя.
-   **Combatant (Боєць):** Поточний стан персонажа в конкретній битві. Зберігає `current_hp`, `current_ammo` тощо. Має зовнішній ключ (`FK`) до шаблону.
-   **Attacks (Атаки):** Довідник усіх можливих бойових атак з базовими параметрами потужності та точності.
-   **Item (Предмети):** Об'єкти, які боєць може використовувати для отримання переваги.
-   **Effect (Ефекти):** Атомарні зміни ігрового стану (наприклад: "отруєння", "посилення на захист", "лікування").
-   **CombatantAttacks & AttackEffects:** Сполучні таблиці для реалізації зв'язків "багато-до-багатьох".
### Опис зв'язків:
1.  **BaseCharacter — Combatant (1:1/1:N):** Один шаблон може бути основою для багатьох бійців (наприклад, армія однакових посіпак), але кожен боєць обов'язково посилається на один шаблон.    
2.  **Combatant — Attacks (N:M):** Реалізовано через таблицю `CombatantAttacks`. Це дозволяє динамічно змінювати набір атак персонажа під час гри.   
3.  **Attacks — Effect (N:M):** Одна атака може накладати кілька ефектів (наприклад, "Удар вогнем" завдає шкоди + накладає ефект "Горіння").  
4.  **Combatant — Item (N:1):** Кожен боєць може тримати в руках один активний предмет у конкретний момент часу. 
5.  **Item — Effect (1:1/1:N):** Кожен предмет прив'язаний до конкретного ефекту, який він активує при використанні.
## Припущення та обмеження
При розробці цієї схеми було зроблено такі припущення:
-   **Динамічні характеристики:** Передбачається, що поточні характеристики бійця (`current_attack`) не можуть перевищувати максимальні значення з `BaseCharacter` без спеціальних ефектів.    
-   **Обмеження інвентарю:** У поточній схемі боєць може мати лише **один активний предмет** одночасно (`item_id` прямо в таблиці `Combatant`).
-   **Стек ефектів:** Ефекти винесені в окрему таблицю, що дозволяє одному і тому самому ефекту (наприклад, "-10 ОЗ") бути частиною як атаки ворога, так і отруєного предмета.
-   **Унікальність атак:** Один і той самий тип атаки (наприклад, "Постріл") може бути призначений різним бійцям з різною кількістю набоїв.
